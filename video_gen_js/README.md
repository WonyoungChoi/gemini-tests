# video_gen_js

Gemini Veo 모델로 동영상을 생성하는 **정적 웹 페이지** 버전입니다. `image_edit_js`와 동일한 패턴이지만, Veo는 비동기 LRO(Long-Running Operation) 호출이라 폴링 단계가 추가됩니다.

## 사용법

1. `video_gen_js/index.html`을 더블 클릭해서 브라우저에서 엽니다 (Chrome / Edge 권장).
2. 좌측에서 **API Key**, **모델**(`veo-3.0-generate-001` 등), **생성 옵션** 을 설정합니다.
   - `사용 가능한 모델 불러오기` 버튼은 `predictLongRunning`을 지원하면서 이름에 `veo` / `video`가 포함된 모델만 필터링합니다.
3. 메인 폼에서 **prompt**(필수), 선택적으로 **입력 이미지**(image-to-video)를 지정한 뒤 **제출**.
4. 작업이 제출되면 `operations/...` ID가 발급되고, 좌측 **폴링 주기 / 최대 대기** 설정에 따라 자동으로 상태를 폴링합니다. 진행 상황은 하단 verbose 로그에 출력됩니다.
5. 완료되면 결과 영역에 `<video>` 태그로 재생 가능한 결과가 표시되고, MIME / 파일 크기 / 가로×세로 / 길이 정보와 다운로드 버튼이 함께 제공됩니다.
6. **중단** 버튼을 누르면 다음 폴링 사이클에 종료됩니다(이미 제출된 작업은 서버에서 계속 과금될 수 있음).

## 옵션

| 옵션               | 값                                                  | 비고                                |
| ------------------ | --------------------------------------------------- | ----------------------------------- |
| Aspect ratio       | `16:9`, `9:16`                                      |                                     |
| Resolution         | (모델 기본값), `720p`, `1080p`                      | 모델이 지원해야 함                  |
| Duration           | 2~10 sec                                            | 모델별 4~8s 권장                    |
| Sample count       | 1~4                                                 | 결과 비디오 개수                    |
| Person generation  | `allow_adult`, `allow_all`, `dont_allow`            | 정책 필터                           |
| Negative prompt    | 자유 입력                                            | 회피하고 싶은 키워드                |
| Seed               | 정수                                                  | 비우면 랜덤                         |

## 제약 / 주의

- **유료 티어 전용**: 무료 키로는 보통 호출 자체가 거부됩니다.
- **응답 시간**: 30초 ~ 수 분 소요. 폴링 주기와 최대 대기 시간을 적절히 설정하세요.
- **CORS**: 응답이 `bytesBase64Encoded`(인라인)면 그대로 표시됩니다. `uri`(GCS 서명 URL)인 경우 브라우저에서 직접 fetch가 CORS로 막힐 수 있어, 자동으로 `URI 새 탭에서 열기` / `다운로드 시도` 폴백 버튼을 보여줍니다.
- **SynthID 워터마크** 가 모든 결과에 삽입됩니다.
- **응답 크기**: 1080p 8초면 수십 MB. 메모리/네트워크 사용량에 주의.
- **쿼터**: 매우 타이트합니다. 429 응답 시 잠시 후 다시 시도.

## 파일 구성

- `app.js` — `predictLongRunning` 호출, 폴링, 응답 파싱, 비디오 렌더링
